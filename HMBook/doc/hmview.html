Help for HMView

HMView is a histogram browser for HMBook. This document describes the usage of the histogram viewer.

The purpose of this program is, to have a quick presentation of your histograms, not to create pretty output. For this you should use the HMBook interpreter HistoMan.

Menu commands:

  • File

    • New Frame
      Create a new window to display several histograms at a time.
    • Load Histogram File
      Loads a histogram file. This command is only visible when the viewer is not started as subprocess. In that case only histograms in the memory are available.
    • Save Histogram File
      Saves a histogram file in HMBook format. It can be reloaded by hmview or HistoMan
    • Export Histogram File
      If you have problems with this viewer you can use PAW to display the histograms written by this command.
    • Quit
      Stop it.
  • Histogram

    • Zoom
      With this option you can change the resolution of the ghostscript interpreter. A larger number magnifies the picture. This button might be slow, since the ghostscript interpreter has to be killed and restarted again. There might be terrible results like core dumps, since there are still some unresolved problems with wxwin when a child process is killed.
      See "Mouse Usage" for how to zoom a range of a histogram.
    • Update
      A timer can be set up for an automatical update.
    • Info
      Some statistical information about the current histogram.
    • Save Current Histogram (*.eps)
      Save this picture as Encapsulated Postscript File. This file can be printed at any postscript printer or can be included, e.g. in TeX documents.
    • Save Current Histogram (*.ascii)
      Save this current histogram as ASCII file.
    • Print Current Histogram
      Print the current histogram via "lpr". To select a different printer you have to set the environment variable "PRINTER" before executing the viewer.
  • Fit
    • User Defined Fit
      Give a function, and the number of paramters for this. You can write a function in C style, with
      "p[0], p[1],..." as parameters and "x" as variable
    • Standarad Fit
      Some predefined functions.
  • Macro
    • Execute Macro
      This command saves the histograms in a temporary file and executes a HistoMan macro file for this. The result is a Postscript file which is presented by calling Ghostview. This might take a while!
  • Help
    • Help
      You found that.
    • About
      I did this.

Mouse usage:

  • Left Mouse Button
    Select a horizontal range for one dimensional histograms. Go to the start point of the desired range, press the left mouse button, select the range and release the mouse button again. The Range is not invert, only the starting point is marked with a line. For two dimensional histograms you can select of course the two dimensional range
  • Middle Mouse Button
    Restore the full range of the histogram, as before any mouse actions. Also useful as "update" button.
  • Right Mouse Button
    Single click: Set the vertical maximum of the current one dimensional histogram.
    Drag region: Select the vertical range for a one dimensional hiostogram.

Select a histogram:

The button "Histogram List" displays a list of all histograms in memory. A double click or a single click and "Ok" selects a histogram. The label of the button is set to the current name of the histogram.

In front of the name is a hint, if the histogram is one dimensional or two dimensional.

Options:

  • Histogram Style
    There are different styles for histogram representation available:
    • 1-dimensional histograms:
      • Histogram:
        Standard representation of a histogram
      • Line:
        Representation as a function, a line connects the bins.
      • Data/Error:
        Representation as datapoints with statistical error.
    • 2-dimensional histograms:
      • Grayscale:
        Each bin is represented by a grayscale rectangle.
      • Lego:
        The histogram is plotted as lego plot or manhattan plot.
      • Scatter:
        The scatter plot is the most ugly choice. In each bin the corresponding number of entries is plot by random dots. Never use this choice to plot a histogram which is filled with a weight. Keep in mind, that the position of each point is not the exact position of an entry!
      • Box:
        Each bin is represented by a rectangular. The size of the rectangle is proportional to the entries for each bin.
      • Surface:
        I like that one. But this might take a while and gives an ugly result, if you have to much bins in each direction!

    Coordinate Display:

    In the upper left corner of the window you see a display of coordinates. If you move the cursor across the histogram, these coordinates correspond to the cross hairs position. The field "Bin" displays the contents of the actual bin. If you choose a 2-dimensional histogram and select a 3-D view like Lego or Surface, the meaning of the cross hairs stays as for a 2-D representation (Gray, Box, Scatter).

    The label of the coordinates are taken from the booking information of the histogram. They might be unreadable sometimes.

    Configuration hints

    This program is writen with the wxWindows toolkit, which is based on MOTIF. So you can use MOTIF resources to change the appearance of your histogram browser:

    For example in your file .Xresources:

    *Form.background:                       grey67
*TransientShell*Dialog.background:      grey67
*Command.background:		        grey77
*MenuButton.background:                 grey77
*SimpleMenu*background:                 grey77
*ScrollbarBackground:                   grey67
*ScrollbarForeground:                   grey37
*Scrollbar*background:                  grey77
*Scrollbar*pointerColor:                black
*Scrollbar*pointerColorBackground:      white
*beNiceToColormap:                      False

*fontList: 	       -*-helvetica-bold-r-normal-*-16-*-*-*-*-*-iso8859-1
*XmTextField.fontList: -*-fixed-bold-r-*-*-15-*-*-*-*-*-iso8859-1
*XmText.fontList:      -*-fixed-bold-r-*-*-15-*-*-*-*-*-iso8859-1
*XmList*fontList:      -*-helvetica-bold-r-normal-*-14-*-*-*-*-*-iso8859-1
